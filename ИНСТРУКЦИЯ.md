# iron-wake — инструкция ко всему

Полное руководство: что умеет бот, как настроить, как пользоваться, как выкатить.
Для архитектуры/деталей — [CLAUDE.md](CLAUDE.md), для деплоя по шагам — [DEPLOY.md](DEPLOY.md).

---

## 1. Что это

Телеграм-бот из двух частей:

1. **Алерты «касание уровня»** — поставь уровень по любому инструменту, бот напишет,
   когда цена его коснётся. Работает по форекс/металлам/нефти/крипте через Yahoo Finance.
2. **Торговый ассистент (VSA + Spring)** — по инструментам с настоящим биржевым объёмом:
   - контекстный анализ (тренд D1 + сильные/слабые уровни + зоны ликвидности + стакан/DOM);
   - авто-сигналы по ложному пробою **Spring** (лонг) и **Upthrust** (шорт) с вход/стоп/цель;
   - трекинг исхода: бот сам сообщает, дошла цена до цели, до стопа или сигнал истёк.

> ⚠️ Это **помощник для решений, а не авто-торговля**. Бот не открывает сделки и не управляет
> риском — он подсвечивает сетапы. Прибыльность зависит от твоей дисциплины и калибровки порогов.

## 2. Настройка `.env`

Скопируй пример и заполни: `cp .env.example .env`

| Переменная | Обязательна? | Зачем |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | **да** | токен бота от @BotFather |
| `OPENROUTER_API_KEY` | да (для текста/AI-разбора) | ключ OpenRouter (openrouter.ai/keys) |
| `OPENROUTER_MODEL` | нет | модель LLM; пусто — берётся дефолт из кода |
| `ADMIN_ID` | для `/broadcast` и `/settings` | твой Telegram ID (узнать — `/myid`) |
| `CCXT_PROXY` | **для BTC/ETH/SOL из РФ** | прокси в разрешённой стране (Binance из РФ даёт 451). См. §6 |
| `BOT_PROXY` | только на сервере из РФ | прокси для связи с Telegram |
| `PAYMENT_TOKEN` | нет | для `/pay` (Telegram Payments) |

## 3. Установка и запуск

```bash
pip install -r requirements.txt
cp .env.example .env      # заполнить токены
python bot.py
```

`bot.db` (база) и `settings.json` (пороги) создаются сами. Тесты движка:

```bash
python tests/test_engine.py     # ожидаем «24/24 тестов прошли»
```

## 4. Команды

**Базовое:**
- `/start` — меню и согласие на обработку данных
- `/help` — справка, `/about` — о боте, `/privacy` — политика
- `/myid` — узнать свой Telegram ID, `/unsubscribe` — отключить уведомления
- `/pay` — оплата (если подключён провайдер), `/broadcast текст` — рассылка (только админ)

**Алерты на уровень:**
- `/alert` — поставить алерт (выбрать инструмент кнопкой или ввести свой тикер Yahoo, потом уровень)
- `/myalerts` — список своих алертов + удаление
- `/cancel` — отменить настройку

**Торговый движок:**
- `/analyze [код]` — анализ инструмента: тренд D1 + сильные/слабые уровни + зоны ликвидности +
  стакан (давление сторон / спред / «стены»). Сверху — человеческий AI-разбор. Без кода — кнопки.
- `/subscribe` — подписка на сигналы по инструменту (кнопки, галочка = подписан)
- `/signals` — последние сигналы со статусом: ⏳ ждём / ✅ цель / 🛑 стоп / ⌛ истёк
- `/settings` — пороги движка кнопками (объём, глубина пробоя) — только админ

---

## 5. Как работает движок (вкратце)

1. Подписываешься на инструмент через `/subscribe`.
2. Раз в час бот пересчитывает контекст (тренд, уровни, зоны) по подписанным инструментам.
3. Каждые 5 минут ищет Spring/Upthrust на свежих H1-свечах. Нашёл — шлёт сигнал подписчикам
   (вход/стоп/цель, приоритет ⭐ если уровень сильный). Один и тот же сигнал не чаще раза в 90 мин.
4. Каждые 5 минут проверяет открытые сигналы: дошла цена до цели → ✅, до стопа → 🛑, за 48 ч
   никуда → ⌛. Сообщает исход.

**Инструменты движка** (есть настоящий биржевой объём):
- Крипта: Bitcoin, Ethereum, Solana (Binance), Toncoin (Bybit)
- Форекс: EUR/USD, GBP/USD, AUD/USD (через **Kraken** — там у фиат-пар реальный объём)

Остальные пары (USD/JPY, USD/CAD, золото, нефть, своя пара) работают только в простых алертах.

## 6. Данные и прокси (важно)

Котировки движка берутся с публичных API бирж через CCXT — **ключи не нужны**. Но:

- **Binance блокирует РФ** (HTTP 451) → для **BTC/ETH/SOL** из России нужен `CCXT_PROXY`
  (прокси в разрешённой стране, `http(s)://` или `socks5://`).
- **Kraken и Bybit РФ не блокируют** → форекс (EUR/USD, GBP/USD, AUD/USD) и TON работают
  **без прокси**. То есть даже без `CCXT_PROXY` ты получишь сигналы по форексу и TON.

Если по инструменту нет данных — в `/analyze` придёт «не удалось получить данные»; проверь прокси.

## 7. Калибровка порогов

Через `/settings` (админ) меняются на лету и сохраняются в `settings.json`:
- `VOL_MULT` — во сколько раз объём свечи должен превышать средний (аномальный объём);
- `BREAK_PCT` — минимальная глубина ложного пробоя;
- `LIQUIDITY_MULT` — порог объёма для зоны ликвидности.

Остальные константы (таймфреймы, окна, интервалы, TTL кеша, срок жизни сигнала 48 ч,
параметры стакана) — в [config.py](config.py).

## 8. Деплой на сервер (Docker)

Бот работает в Docker-контейнере `iron-wake-bot` (не systemd). Полная процедура с откатом —
[DEPLOY.md](DEPLOY.md). Кратко:

1. **Важно:** в этом релизе появились новые модули (`config.py`, `data_fetcher.py`,
   `analyzer.py`, `pattern_detector.py`, `scheduler.py`). `Dockerfile.bot` копирует файлы
   **поимённо** — допиши их в строку `COPY ... ./`, иначе бот упадёт с `ModuleNotFoundError`.
2. Бэкап базы: `cp -p bot.db "bot.db.bak-$(date +%Y%m%d)"` (в релизе миграции БД — обязательно).
3. Забрать код: `cd /opt/apps/bot && git pull --ff-only origin main`.
4. Пересобрать образ и перезапустить контейнер `iron-wake-bot` (точные команды `docker compose`
   — в [DEPLOY.md](DEPLOY.md)). Зависимости `ccxt`/`numpy` подтянутся при сборке автоматически.
5. При необходимости добавить `CCXT_PROXY` в `.env` (для BTC/ETH/SOL из РФ; форекс и TON
   работают и без него).
6. Логи: `docker logs --tail 80 iron-wake-bot` — без `Traceback`.

Новые таблицы (`levels`, `signals`, `subscriptions`) и колонка `signals.bar_time` создаются
автоматически при старте. Старые данные и алерты не затрагиваются.

---

## 9. Проверка после деплоя

- В логах: `[check_alerts] активных алертов: N` и `[run_analysis] ...` без `Traceback`.
- В Telegram: `/start` отвечает; `/alert` ставит алерт; `/subscribe` → BTC или EUR/USD;
  `/analyze EURUSD` показывает тренд + уровни + стакан (форекс через Kraken работает без прокси).
- `/signals` показывает сигналы со статусами по мере появления.
