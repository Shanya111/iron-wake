# Инструкция по деплою iron-wake

Деплой из GitHub (`git@github.com:Shanya111/iron-wake.git`, ветка `main`) на сервер.
Бот — это сервис, который ходит к Telegram и OpenRouter через серверный прокси
(`BOT_PROXY` / `trust_env`), и к Yahoo Finance за котировками.

## Что в этом релизе (важно знать перед деплоем)

1. **Миграция БД (выполняется автоматически при старте).** Таблица `alerts` меняет схему:
   добавлена колонка `pair` (инструмент), `start_above` стал nullable, убрано ограничение
   «один алерт на пользователя», колонка `direction` удалена. `init_db()` при старте сам
   перестроит таблицу и перенесёт существующие алерты — все старые алерты станут `USDJPY`.
   **Руками ничего делать не нужно**, но перед стартом нужен бэкап `bot.db` (см. ниже).
2. **Новая зависимость — `pandas`** (добавлена в `requirements.txt`). Нужен `pip install`.
3. Логика алертов: теперь «касание уровня» вместо «выше/ниже», много пар на пользователя,
   10 инструментов + своя пара. На инфраструктуру не влияет.

`bot.db` и `.env` — в `.gitignore`, деплой их **не трогает**: на сервере остаются свои
файлы с данными пользователей и секретами.

## Шаги деплоя

> Подставь свои значения: `<КАТАЛОГ>` — папка проекта на сервере, `<СЕРВИС>` — имя systemd-сервиса
> (узнать: `systemctl list-units | grep -i iron` или как настроено у тебя), `<VENV>` — путь к venv.

```bash
# 0. Перейти в каталог проекта
cd <КАТАЛОГ>

# 1. БЭКАП базы (обязательно — единственный необратимый шаг этого релиза)
cp bot.db "bot.db.bak.$(date +%Y%m%d-%H%M%S)"

# 2. Забрать код
git fetch origin
git checkout main
git pull --ff-only origin main

# 3. Обновить зависимости (появился pandas)
<VENV>/bin/pip install -r requirements.txt

# 4. Перезапустить сервис (миграция БД пройдёт автоматически на старте)
sudo systemctl restart <СЕРВИС>

# 5. Проверить логи: старт без ошибок + проверка алертов идёт
journalctl -u <СЕРВИС> -n 80 --no-pager
```

## Проверка после деплоя

- В логах есть строки `[check_alerts] активных алертов: N` и по парам — без `Traceback`.
- В Telegram: `/start` отвечает; `/alert` показывает кнопки инструментов; выбор инструмента
  показывает актуальную цену; `/myalerts` показывает существующие алерты.
- Если у пользователей были старые алерты — они на месте и помечены как USD/JPY.

## Откат (если что-то пошло не так)

```bash
cd <КАТАЛОГ>
sudo systemctl stop <СЕРВИС>

# вернуть код на предыдущий коммит
git log --oneline -n 5          # найти предыдущий рабочий коммит <SHA>
git checkout <SHA>

# восстановить базу из бэкапа (имя — из шага 1)
cp "bot.db.bak.ГГГГММДД-ЧЧММСС" bot.db

sudo systemctl start <СЕРВИС>
journalctl -u <СЕРВИС> -n 50 --no-pager
```

## Примечания

- Кратковременный простой на время рестарта — норма. Telegram поставит апдейты в очередь и
  отдаст после старта, сообщения не теряются.
- Yahoo Finance не любит частые запросы. Бот запрашивает только инструменты, на которые есть
  активные алерты (1 запрос на пару раз в 5 минут). Если в логах начнут мелькать `429` —
  значит пар стало слишком много; тогда переходим на батч-запрос (отметка в плане проекта).
